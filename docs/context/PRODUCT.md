# PRODUCT.md

このファイルは、まいカゴのプロダクト判断を復元するための永続コンテキストです。新機能、仕様変更、UI変更、課金・広告・共有機能に関わる作業では、実装前にこのファイルを読んでください。

## Product Summary

**まいカゴ** は、買い物中の「買いすぎ」と「合計金額が分からない不安」を減らすための買い物リスト管理アプリです。

ユーザーは商品名、価格、個数、割引率、予算を管理しながら、買い物中にリアルタイムで合計金額を確認できます。カメラOCR、レシピ取り込み、クラウド同期、共有タブ、テーマ変更、プレミアム買い切りなどを備えた Flutter マルチプラットフォームアプリです。

## Target Users

- スーパーやドラッグストアなどで、買い物中に合計金額を把握したい人
- 予算内に収めたい人、買いすぎを防ぎたい人
- メモアプリと電卓を行き来するのが面倒な人
- 家族や同居人と買い物リストを共有したい人
- ログインせずにまず試したい人

## Core User Value

最重要価値は **買い物中に、今いくら使う予定なのかをすぐ分かること** です。

機能追加やUI変更では、次の価値を優先してください。

1. 合計金額が分かりやすい
2. 商品追加・編集が速い
3. 買い物中でも迷わず操作できる
4. 予算超過や入力ミスに気づきやすい
5. ゲストでもログイン済みでも体験が破綻しない
6. 共有・同期してもリストの信頼性が保たれる

## Core Workflows

### 買い物リスト作成

ユーザーはタブまたは店舗ごとのリストを開き、商品名、価格、個数、割引率を入力します。合計金額は即時に反映されます。

### 買い物中の確認

ユーザーは買い物中に商品を追加・編集・購入済みにしながら、画面下部の合計や予算状態を確認します。操作は片手でも行いやすいことを重視します。

### カメラOCR

ユーザーは値札を撮影し、OCR/AI解析で商品名・税込価格を抽出します。解析結果は確認画面を経由して、ユーザーが納得してからリストに追加します。

### レシピ取り込み

ユーザーはレシピURLから材料を取り込み、買い物リストにまとめて追加します。解析結果は確認・編集できる必要があります。

### 同期・共有

Googleログイン時は Firestore で複数デバイス同期します。共有タブでは家族などと共同編集できます。同期中でもUIは重くならず、ユーザーの操作を妨げないことを優先します。

### プレミアム

買い切り課金により、広告非表示、リスト・タブ無制限などの機能制限解除を提供します。マネタイズは買い物体験を邪魔しない範囲に抑えます。

## Product Principles

- **実用性優先:** 見た目の派手さより、買い物中の読みやすさ・操作しやすさを優先する。
- **即時反応:** 商品追加、編集、購入済み切り替え、予算反映はできるだけ即時に見えるようにする。
- **確認可能:** OCR、レシピ解析、共有、削除、課金など失敗や誤操作の影響が大きい機能は確認・復旧導線を持つ。
- **ゲスト尊重:** ゲストモードでも主要な買い物リスト体験を成立させる。
- **同期の信頼性:** クラウド同期では、データ消失や重複、意図しない上書きを避ける。
- **初心者にやさしい:** エラーや制限は専門用語を避け、次に何をすればよいか分かる文言にする。
- **一貫したUI:** UI変更時は `DESIGN.md` を読み、既存テーマと共通コンポーネントに合わせる。

## Non-Goals

現時点では、次の方向に過度に広げないでください。

- 汎用的な家計簿アプリ化
- 在庫管理、栄養管理、献立管理の本格化
- ECサイトや決済アプリ化
- 複雑なSNS、チャット、コミュニティ機能
- 買い物以外のToDo管理アプリ化
- 高度な業務用POS・店舗管理機能

これらに近い提案が出た場合は、買い物中の合計金額確認という中心価値に戻して必要性を判断します。

## Feature Decision Checklist

新機能や仕様変更では、実装前に次を確認してください。

- この変更は「買いすぎ防止」または「買い物中の操作効率」に直接効くか
- ゲストモード、ログイン済み、プレミアム状態で矛盾しないか
- 共有タブやリアルタイム同期で破綻しないか
- OCR/AI解析結果をユーザーが確認・修正できるか
- エラー時にデータが失われないか
- UIは `DESIGN.md` と既存コンポーネントに沿っているか
- 検証方法は `TESTING.md` または既存テスト方針で説明できるか

## Product Copy Guidelines

- 文言は日本語を基本にする。
- 初心者向けに、専門用語より行動が分かる言葉を使う。
- エラーは「何が起きたか」と「次に何をすればいいか」を短く示す。
- 課金・広告・共有・ログインに関わる文言は、誤解がないように具体的にする。
- 買い物中に読む文言は短くする。

## Source of Truth

- バージョン番号は `pubspec.yaml` の `version` を正とする。
- 実装済みの機能・画面構成は既存コードを正とする。
- UI方針は `DESIGN.md` と既存 `SettingsTheme` / `CommonDialog` を正とする。
- アーキテクチャ方針は `ARCHITECTURE.md` と既存ディレクトリ構成を正とする。
